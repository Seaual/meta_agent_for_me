# CONVENTIONS.md — Code Review Enforcer Teams v2 规范

> 本文件定义 Team 的代码规范和审查标准

---

## 文件命名规范

| 类型 | 规范 | 示例 |
|-----|------|------|
| Agent 文件 | kebab-case，与 `name` 字段一致 | `code-reviewer.md` |
| 输出报告 | 固定名称 | `review-report.md` |

---

## YAML Frontmatter 规范

每个 agent 的 frontmatter 必须包含：

```yaml
---
name: kebab-case-name
description: |
  Activate when [动词短语].
  Handles: [场景A], [场景B].
  Keywords: [英文词], [中文词].
  Do NOT use for: [排除场景].
allowed-tools: Read, Grep, Glob, Bash, Write
---
```

---

## 工具权限规范

| 工具 | 用途 | 风险 |
|-----|------|------|
| `Read` | 读取文件内容 | 最低 |
| `Grep` | 搜索模式 | 最低 |
| `Glob` | 文件匹配 | 最低 |
| `Bash` | Git 只读命令 + Python AST | 高，仅限白名单命令 |
| `Write` | 写入 review-report.md | 中，仅限指定文件 |

## 代码规范

### Python 类型注解

```python
# 推荐：所有公共函数添加类型注解
def process_data(data: str) -> str:
    """处理数据，返回清洗后的字符串。"""
    return data.strip()

# 私有方法可选择性添加
def _internal_helper(value: int) -> int:
    return value * 2
```

### 命名规范

- 变量: `snake_case`
- 函数: `snake_case`
- 类: `PascalCase`
- 常量: `UPPER_SNAKE_CASE`

### 行长度

- 最大 100 字符
- URL 可例外

### 异常处理

```python
# 正确：指定具体异常类型
try:
    process()
except ValueError as e:
    handle_error(e)

# 错误：裸 except
try:
    process()
except:  # Critical: 未指定异常类型
    pass
```

```markdown

```

|-------|------|
| Critical | {n} |
| Warning | {n} |

### {filename}

| 行号 | 规则 | 严重性 | 描述 |
|-----|------|--------|------|

### Critical（必须修复）
...

### Warning（建议修复）

### 严重性定义

| 级别 | 含义 | 处理建议 |
|-----|------|---------|
| Critical | 可能导致运行时错误 | 必须修复 |
| Warning | 影响代码质量 | 建议修复 |
| Info | 提升可读性 | 可选修复 |

## Bash 命令白名单

### 允许的 Git 命令（只读）

```bash
git diff --name-only HEAD~1 HEAD -- '*.py'
git diff --cached --name-only -- '*.py'
git rev-parse --abbrev-ref HEAD
git rev-parse --show-toplevel
git log -1 --format='%H %s'
git rev-parse --is-inside-work-tree
```

### 允许的 Python 命令

```bash
python3 -c "..." -- "$filepath"
```

### 禁止的命令

- `rm -rf *`
- `git push`
- `git reset --hard`
- 任何写入源文件的命令

---

## 安全红线

1. 不修改源代码文件
2. 只写入 `review-report.md`
3. Bash 仅执行白名单命令
4. 不硬编码凭证
5. 不执行网络请求
