## 新增内容

### 1. 类型注解检查规则 [Warning]

检测函数参数和返回值是否有类型注解。

**检测逻辑**:
- 使用 Python AST 解析
- 检查参数注解（`param: type`）
- 检查返回值注解（`-> type`）

**排除项**:
- `__init__` 方法（可选注解）
- 私有方法（`_` 开头）可选择性跳过
- 魔术方法
- `self` 和 `cls` 参数

**示例**:

```python
# 问题代码 [Warning]
def process_data(data):  # 缺少类型注解
    return data.strip()

# 正确写法
def process_data(data: str) -> str:
    return data.strip()
```

### 2. ASCII 严重性分布图

在审查报告的统计摘要部分增加可视化图表：

```markdown
**严重性分布图**:
```
Critical [==] 2
Warning  [=====] 5
Info     [===] 3
```
```

**实现规范**:
- 最大宽度: 10 字符
- 填充字符: `=`
- 名称宽度: 8 字符（左对齐）

### 技术决策

**类型注解检测方式**: 选择 AST 解析而非正则

| 方案 | 优点 | 缺点 | 决策 |
|-----|------|------|------|
| 正则 | 简单快速 | 无法处理复杂类型、误报高 | 否 |
| AST | 准确、支持复杂类型 | 需要 Python 环境 | **采用** |
