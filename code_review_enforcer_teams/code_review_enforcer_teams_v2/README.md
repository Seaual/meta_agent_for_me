## v2 规则集

| 序号 | 规则名称 | 严重性 | 检测内容 | 变更说明 |
|-----|---------|--------|---------|---------|
| 1 | 裸 except | Critical | `except:` 未指定异常类型 | 不变 |
| 2 | 未处理异常 | Critical | try 块外可能抛出异常未捕获 | 不变 |
| 3 | 过长行 | Warning | 单行超过 **100** 字符 | 阈值从 120 改为 100 |
| 4 | 未使用 import | Warning | 导入模块但未使用 | 不变 |
| 5 | 变量命名 | Warning | 非 snake_case 变量名 | 不变 |
| 6 | 函数命名 | Warning | 非 snake_case 函数名 | 不变 |
| 7 | 类型注解检查 | Warning | 函数参数/返回值缺少类型注解 | **新增** |
| 8 | 缺少 docstring | Info | 函数/类/模块缺少文档字符串 | 不变 |

**已删除**: Magic Number 检查（v1 规则 7），误报率过高

## v2 新增功能

### 1. 类型注解检查

检测函数参数和返回值是否有类型注解，排除私有方法和魔术方法。

```python
# 问题代码
def process_data(data):  # Warning: 缺少类型注解
    return data.strip()

# 正确写法
def process_data(data: str) -> str:
    return data.strip()
```

### 2. 严重性分级统计图

审查报告新增 ASCII 柱状图：

```
**严重性分布图**:
Critical [==] 2
Warning  [=====] 5
Info     [===] 3
```

### 3. 更严格的行长度限制

行长度阈值从 120 字符调整为 100 字符。
