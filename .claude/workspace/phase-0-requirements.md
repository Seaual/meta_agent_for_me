# Phase 0 需求文档

## 1. 核心目标

创建一个**题目生成 Agent Team**，用于检验用户对 PDF 文档内容的理解程度。

## 2. 输入输出

| 项目 | 说明 |
|------|------|
| **输入** | `input/` 目录下的 PDF 文件 |
| **输出** | Markdown 文件（题目 + 答案 + 解析）|
| **导出** | Word 格式导出功能 |

## 3. 题目类型

- **单选题**：4 个选项，1 个正确答案
- **多选题**：4+ 个选项，2+ 个正确答案
- **判断题**：正确/错误，附理由

## 4. 业务场景

| 项目 | 说明 |
|------|------|
| **领域** | 招聘/培训资料（行车调度室调度员岗位公开招聘复习资料）|
| **用途** | 知识检验 — 检验对文档内容的理解 |
| **语言** | 中文输出 |

## 5. 运行模式

| 项目 | 说明 |
|------|------|
| **数量控制** | 用户在对话中动态指定（如"生成5道单选题"）|
| **PDF 来源** | 仅本地 input 目录 |
| **外部服务** | 无 |
| **MCP 工具** | 无 |

## 6. 自我改进

| 功能 | 启用状态 |
|------|---------|
| **Self-Improving** | ✅ 启用 |
| **Instincts 持续学习** | ✅ 启用（两层 .learnings/ 结构）|

## 7. Team 规格

| 项目 | 选择 |
|------|------|
| **规模** | 标准型（4-5 个 agent）|
| **角色分工** | 由系统设计 |
| **运行时 Profile** | minimal（仅基础安全检查）|

## 8. 版本信息

- **Team 名称**：question-generator
- **起始版本**：v1
