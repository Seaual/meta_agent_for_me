# Critical Analysis — question-generator

## 1. 假设挑战

| 假设 | 挑战 | 风险等级 |
|------|------|---------|
| "PDF 内容可被正确解析" | 扫描版 PDF、图片 PDF 无法提取文字 | 中 |
| "AI 生成的题目足够准确" | 可能出现事实错误、答案不唯一的情况 | 高 |
| "用户会检查并确认题目" | 用户可能直接使用未审查的题目 | 中 |

## 2. 简化建议

### 方案 A：合并 question-generator + quality-reviewer

**理由**：让生成 agent 自带质量检查，减少交互步骤。

**风险**：无法独立审查，可能遗漏问题。

**建议**：**不采纳**。独立审查能发现生成者视角的盲点。

### 方案 B：省略 word-exporter

**理由**：Markdown 文件可直接用 Typora 等工具导出 Word。

**风险**：增加用户操作成本。

**建议**：**保留**。Word 导出是明确需求，自动化能提升用户体验。

### 方案 C：最小可行方案（2 agents）

```
pdf-reader → question-generator（含质量自查）
```

**理由**：快速交付核心价值，后续迭代增加功能。

**建议**：**可作为降级方案**。如果时间紧迫，先实现核心功能。

## 3. 边界条件

### 必须处理的情况

| 场景 | 处理方式 |
|------|---------|
| PDF 文件不存在 | 明确提示用户放入 input 目录 |
| PDF 内容为空 | 提示用户检查文件有效性 |
| 用户未指定题目数量 | 默认每种类型 3 道 |
| PDF 页数过多（>50页）| 提示用户可能耗时较长 |
| Word 导出失败 | 提供降级方案（输出 Markdown）|

### 题目生成约束

- 单选题：必须有且仅有 1 个正确答案
- 多选题：必须有 2+ 个正确答案，且不能全选
- 判断题：答案必须是"正确"或"错误"
- 所有题目：必须有解析说明答案理由

## 4. 质量风险

| 问题 | 检测方式 | 修复方式 |
|------|---------|---------|
| 答案与解析矛盾 | quality-reviewer 检查 | 返工重生成 |
| 题目内容与 PDF 无关 | 交叉验证 | 返工重生成 |
| 选项明显不合理（如"以上都对"）| 规则检查 | 返工重生成 |
| 题目重复 | 去重检查 | 合并或删除 |

## 5. 关键风险

**最高风险**：AI 生成的题目可能出现事实错误。

**缓解措施**：
1. quality-reviewer 必须对比原文验证答案
2. self-improving 记录错误类型，避免重复
3. 提示用户最终确认

## 6. Agent 数量建议

**推荐**：4 个 agent

| Agent | 必要性 |
|-------|--------|
| pdf-reader | ✅ 必需 |
| question-generator | ✅ 必需 |
| quality-reviewer | ✅ 必需（质量保障）|
| word-exporter | ⚠️ 可选（但需求明确要求）|

不建议增加到 5 个，已足够覆盖需求。
